## Common Questions

### Project has a README but no AGENTS.md

Keep README.md for humans. Create AGENTS.md for AI with technical details, quick commands, and pattern references.

### Project has scattered documentation

Create `docs/INDEX.md` linking to existing docs. Don't move files initially â€” consolidate gradually as you update them.

### Project has undocumented patterns

Start with AGENTS.md covering main patterns. Create rule files as you encounter AI generating inconsistent code.

### Using different folder structure

Document YOUR structure, don't change it. Use rules to tell AI where things go in your project. Adapt templates to match your conventions.

### Team has different tooling

The methodology is tool-agnostic. Adapt rule file format:
- **Cursor**: `.cursor/rules/*.mdc`
- **Claude Code**: `CLAUDE.md`
- **Copilot**: `.github/copilot-instructions.md`
- **Other**: Standard `.md` files

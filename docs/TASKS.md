## How to Use This File

### Adding Tasks

```markdown
- [ ] **PREFIX-XXX**: Brief task description
  - Priority: P0/P1/P2/P3
  - Estimate: X hours (optional)
  - Depends on: OTHER-ID (optional)
  - Files: `path/to/file.ts` (optional)
```

### Priority Levels

| Level | Meaning | Examples |
|-------|---------|----------|
| P0 | Critical/Blocker | Core functionality, security |
| P1 | High | Key features, major bugs |
| P2 | Medium | Improvements, minor bugs |
| P3 | Low | Nice-to-have, polish |

### Updating Progress

1. Mark task as in progress: Change `- [ ]` to `- [~]` (optional)
2. Mark task as done: Change `- [ ]` to `- [x]`
3. Update feature progress in summary table
4. Move completed tasks to "Completed Tasks" section
